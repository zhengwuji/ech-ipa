# 自动推送使用说明

## 自动推送代码到GitHub

每次修改代码后，需要推送到GitHub时，有以下几种方式：

### 方法一：使用自动推送脚本（推荐）

双击运行 `setup_auto_push.bat` 或在命令行执行：
```bash
python auto_commit_push.py
```

这个脚本会：
1. 自动递增版本号（1.0.00 -> 1.0.01 -> ... -> 9.9.99 -> 1.0.00）
2. 自动添加所有修改的文件
3. 自动提交更改（无需确认）
4. 强制推送到GitHub

### 方法二：手动推送

如果需要手动控制，可以使用：
```bash
git add .
git commit -m "你的提交信息"
git push --force origin main
```

## GitHub自动构建

推送代码到GitHub后：
1. GitHub Actions 会自动检测到推送
2. 自动构建 Windows EXE 文件
3. 自动发布到 Releases，版本号自动递增
4. 可以从 Releases 页面下载最新版本

## 版本号说明

- 格式：主版本.次版本.修订版本（如 1.0.00）
- 范围：1.0.00 到 9.9.99
- 循环：9.9.99 后自动回到 1.0.00
- 每次推送自动递增修订版本号
